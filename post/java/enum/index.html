Java 枚举类型

Feb 7, 2023

枚举类型（enum type）是由一组具有相同含义的常量组合而成

在引入枚举类型之前，通常用一组 int 常量表示每种类型。

public class Color {
    private static final int RED = 1;
    private static final int GREEN = 2;
    private static final int BLACK = 3;
}

用 int 常量表示的方法不具有类型安全性。方法只能接收 int 类型，不同 int 值表示不同的 Color，此时无法严格控制入参，很容易出现类型错误。

public void color(int color) {
    // do something
}

Java SE 1.5 版本引用了枚举类型，使用 enum 关键字修饰。在编译期间，编译器会对枚举类型进行修改和扩展。

枚举类型隐式使用 public static final 修饰（编译器修改），按照 Java 命名规范，所有名称都应该采用大写的形式。

public enum Color {
    RED, GREEN, BLACK;
}

枚举类型可以额外增加属性，只需声明成员属性，并添加具有所有属性的构造器。

枚举类型是不可变的，因此，推荐使用 final 和 private 修饰成员属性。

public enum Color {
    RED("#FF0000", "红色"),
    GREEN("#008000", "绿色"),
    BLACK("#000000", "黑色");

    private final String rgb;
    private final String name;

    Color(String rgb, String name) {
        this.rgb = rgb;
        this.name = name;
    }
}

可以用 values() 方法获取到当前枚举类型的所有值，会按照声明的顺序返回数组。

Color[] values = Color.values();
for (Color color : values) {
    System.out.println(color);
}

方法绑定

当我们需要将不同的方法与每个常量关联起来时，常见的做法是使用 switch 进行判断。

当在枚举中添加新的类型时，很容易忘记在 color 方法中添加新的分支。

public void color(Color color) {
    switch (color) {
        case RED -> System.out.println("red");
        case GREEN -> System.out.println("green");
        case BLACK -> System.out.println("black");
    }
    throw new  IllegalArgumentException("Unknown color");
}

Java SE 1.6 版本引入了类主体（class body），可以将不同方法与枚举关联起来。

当枚举中添加新的类型而不添加类主体，编译器会拒绝编译。

enum Operation {
    PLUS { double eval(double x, double y) { return x + y; } },
    MINUS { double eval(double x, double y) { return x - y; } },
    TIMES { double eval(double x, double y) { return x * y; } },
    DIVIDED_BY { double eval(double x, double y) { return x / y; } };

    // Each constant supports an arithmetic operation
    abstract double eval(double x, double y);
}

编译后每个类主体都会生成一个独立类。

Operation$1.class
Operation$2.class
Operation$3.class
Operation$4.class

生成的类文件反编译后。

enum Operation$1 {
    private Operation$1() {
    }

    double eval(double var1, double var3) {
        return var1 + var3;
    }
}

Java 8 之后可以进一步结合 lambda。

import java.util.function.BiFunction;

enum Operation {

    ADD((x, y) -> x + y),
    SUBTRACT((x, y) -> x - y),
    MULTIPLY((x, y) -> x * y);

    private final BiFunction<Integer, Integer, Integer> operation;

    Operation(BiFunction<Integer, Integer, Integer> operation) {
        this.operation = operation;
    }

    public int apply(int x, int y) {
        return operation.apply(x, y);
    }
}

Enum 类

Java 中所有的枚举类型都会继承 java.lang.Enum 类。

public abstract class Enum<E extends Enum<E>>
        implements Constable, Comparable<E>, Serializable

总结

  1. 枚举类型没有可以访问的构造器，因此，不可以实例化。
  2. 枚举类型不可继承，因为枚举类型默认会继承 Enum 类，且 Java 只支持单继承。
  3. 枚举类型内可以添加额外的方法，扩招功能。
