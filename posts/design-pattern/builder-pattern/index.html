设计模式-建造者模式

2023-07-14

建造者模式关注如何创建对象，将对象的创建和使用进行分离，使用者不需要关心对象的创建细节。

单例模式

饿汉式

根据 JLS 8.3.1.1 可知类初始化时就会创建该单例对象。

静态变量

public class Singleton {
    private final static Singleton SINGLETON = new Singleton();

    private Singleton() {
    }

    public static Singleton getInstance() {
        return SINGLETON;
    }
}

静态方法块

public class Singleton {
    private static Singleton singleton;

    static {
        singleton = new Singleton();
    }

    private Singleton() {
    }

    public static Singleton getInstance() {
        return singleton;
    }
}

静态变量会在编译时收集起来，收集到静态方法块中初始化。

通过 javap -v 反编译字节码文件，可以发现上边两段代码在字节码层面是一致的。

懒汉式

类加载时不会创建该单例对象，首次使用的时候才会创建此对象。

双重检查

使用 Double-checked locking 实现线程安全的单例模式。

public class Singleton {
    // 添加 volatile 防止指令重排序
    private volatile static Singleton singleton;

    private Singleton() {
    }

    public static Singleton getInstance() {
        if (singleton == null) {
            synchronized (singleton) {
                if (singleton == null) {
                    singleton = new Singleton();
                }
            }
        }
        return singleton;
    }
}

静态内部类

外部类加载时，不会直接加载内部类，当第一次使用内部类时才会加载类。

public class Singleton {

    private Singleton() {
    }

    private static final class SingletonHolder {
        static final Singleton singleton = new Singleton();
    }

    public static Singleton getInstance() {
        return SingletonHolder.singleton;
    }
}

枚举类单例

使用反射和序列化可以破坏单例模式，推荐使用枚举类实现单例模式。

public enum Singleton {
    SINGLETON;
    public void f() {
    }
}
